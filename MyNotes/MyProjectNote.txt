Go to the directory where the project's root folder will be created.
Create the projec't root folder manually, and open it using JetBrain IntelliJ IDEA.

☑️--------------------------------------------------

Run the command [npm init] into the Terminal, to initialize the project,
then provide a package name,
then provide a description,
then provide a author name,
then answer other questions as default, and the project initializatio is done.

☑️--------------------------------------------------

Now run the command [npm i express] in the Terminal to install ExpressJS as main dependencies,
then install this package [npm i -D @types/express] as DevDependencies to support TypeScript.

☑️--------------------------------------------------

Now run the command [npm i node] in the Terminal to install NodeJS as main dependencies,
then install this package [npm i -D @types/node] as DevDependencies to support TypeScript.

☑️--------------------------------------------------

Now run the command [npm i -D typescript] in the Terminal to install TypeScript as DevDependencies,
then close and reopen the IDE and then run this command [tsc --init] to initialize TypeScript and generate tsconfig.json file.

Create two folders in the root directory, 'dist' and 'src'.
Create another folder 'app' inside the 'src' folder.
Create two TypeScript files, 'app.ts' and 'server.ts' inside the 'src' folder.

Open the tsconfig.json file,
Find out [ // "rootDir": "./", ], uncomment it, and set the root directory to [ "rootDir": "./src/", ].
Find out [ // "outDir": "./", ], uncomment it, and set the output directory to [ "outDir": "./dist/", ].

Now go to the Terminal, run the [ tsc ] command.
It will compile .ts files from the src folder and will create respective converted .js files in the dist folder.

If everything works fine, then run the command [tsc -w] in the Terminal.
It will watch all .ts files for any changes and will immediately create respective converted .js files in the dist folder.

☑️--------------------------------------------------

Now run the command [npm i -D ts-node-dev] in the Termina, to install the package ts-node-dev,
then open the package.json file and add [ "dev": "ts-node-dev --respawn --transpile-only src/server.ts" ] in the 'scripts' array,
then also add [ "build": "tsc" ] in the 'scripts' array.

☑️--------------------------------------------------

Then add a run configuration in the IntelliJ IDEA targeting 'npm run dev' command. Add 'Compile TypeScript' in the 'Before launch' section.
From now on, we don't need to run 'tsc -w' and nodemon anymore.
This 'ts-node-dev' package will do all necessary compilation and will run the project directly from './src/server.ts' file.

MVC Pattern === Models + Views + Controllers
Nowadays 'Views' are mostly generated in front-end using React or something similar. So we'll do only 'Models' and 'Controllers'.


project-root/
├── src/                                 # প্রোজেক্টের সমস্ত সোর্স কোড
│   ├── app/                             # Express অ্যাপ্লিকেশনের মূল লজিক
│   │   ├── config/                      # কনফিগারেশন ফাইল
│   │   │   └── config.ts                # এনভায়রনমেন্ট ভেরিয়েবল, ডেটাবেস কনফিগারেশন
│   │   ├── middlewares/                 # কাস্টম Express মিডলওয়্যার ফাংশন
│   │   │   ├── auth.middleware.ts       # অথেন্টিকেশন সংক্রান্ত মিডলওয়্যার
│   │   │   └── error.middleware.ts      # এরর হ্যান্ডলিং মিডলওয়্যার
│   │   ├── models/                      # ডেটাবেস স্কিমা এবং মডেল
│   │   │   ├── User.model.ts            # ইউজার মডেল (ডাটাবেস স্কিমা)
│   │   │   └── Product.model.ts         # প্রোডাক্ট মডেল
│   │   ├── controllers/                 # রিকোয়েস্ট হ্যান্ডলিং লজিক
│   │   │   ├── user.controller.ts       # ইউজার সম্পর্কিত লজিক
│   │   │   └── product.controller.ts    # প্রোডাক্ট সম্পর্কিত লজিক
│   │   ├── routes/                      # API এন্ডপয়েন্ট রুট
│   │   │   ├── user.routes.ts           # ইউজার সম্পর্কিত API রুট
│   │   │   └── product.routes.ts        # প্রোডাক্ট সম্পর্কিত API রুট
│   │   ├── services/                    # বিজনেস লজিক এবং ডেটা প্রসেসিং
│   │   │   ├── auth.service.ts          # অথেন্টিকেশন সার্ভিস
│   │   │   └── product.service.ts       # প্রোডাক্ট সার্ভিস
│   │   ├── interfaces/                  # TypeScript ইন্টারফেস এবং কাস্টম টাইপস
│   │   │   ├── express.d.ts             # Express এর জন্য কাস্টম টাইপ ডিক্লারেশন (যদি লাগে)
│   │   │   ├── user.interface.ts        # ইউজার সম্পর্কিত ইন্টারফেস
│   │   │   └── product.interface.ts     # প্রোডাক্ট সম্পর্কিত ইন্টারফেস
│   │   └── utils/                       # সাধারণ ইউটিলিটি ফাংশন
│   │       ├── jwt.utils.ts             # JWT হেল্পার ফাংশন
│   │       └── validators.ts            # ভ্যালিডেশন ফাংশন
│   ├── app.ts                           # Express অ্যাপ ইনিশিয়ালাইজেশন
│   └── server.ts                        # অ্যাপ স্টার্টআপ ফাইল
├── .env                                 # এনভায়রনমেন্ট ভেরিয়েবল
├── .gitignore                           # গিট রিপোজিটরি থেকে বাদ দিতে চান এমন ফাইল
├── package.json                         # প্রোজেক্ট মেটাডেটা এবং ডিপেন্ডেন্সি
└── tsconfig.json                        # TypeScript কম্পাইলার কনফিগারেশন

Pino:
Run the command [ npm install pino @types/pino ] in the Terminal to install Pino with TypeScript support.
Create a new file name 'pinoConfig.ts' in the directory src/config/pinoConfig.ts and write configuration code, and export it.
Run the command [ npm install pino-pretty --save-dev ] in the Terminal to install pino-pretty, which will make human-readable logs in development.
In the package.json file, in the scripts array, include [ | npx pino-pretty] at the end of the "dev" command.
Run the command [ npm install pino-http @types/pino-http ] in the Terminal to install pino-http, for request loggin while using ExpressJS.
Now in the pinoConfig.ts file import [ import pinoHttp from 'pino-http'; ], export it too.
Now in the app.ts file, import [ import {logger, pinoHttp} from './config/pinoConfig'; ] and include [ app.use(pinoHttp({logger})); ].
Now req can be logged like [ req.log.info('Logging GET request using PINO.'); ]
